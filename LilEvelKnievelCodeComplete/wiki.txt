<u>Instructions</u><br />
Lil Evel Knievel V1.0 game play is very open and simple due to the aggressive coding competition deadline: jump over varying platform gaps and pull off cool skateboard flips and trick combos to complete each level.
<br /><br />  
There are four jump categories on ground: Skip, Jump, Leap, Hurl.  Once airborne, Lil EK can swap direction or flip somersault.  Move opposite direction while airborne inverts somersault and down to accelerate land.

<u>Audio</u><br />
Lil Evel Knievel includes digitized sound extensively by leveraging the <a href="https://github.com/maxim-zhao/pcmenc">pcmenc</a> tool and uses development techniques outlined in devkitSMS Programming <a href="http://steveproxna.blogspot.com/2022/07/devkitsms-programming-sample-v.html">Sample</a> under PCM Samples section.  Some riffs you may have missed include: <i>AC/DC, Austin Powers, Gn'R, Simpsons, Van Halen, Wayne's World, WWF Wrestling</i>.
<br><br>
Music is often easy to neglect in a time pressure coding competition but Lil EK has been <a href="https://www.youtube.com/watch?v=wtaDY79qoK4&t=1000s">quoted</a> as having "<b><i>perhaps the single most impressive audo soundtrack the little PSG has ever seen</i></b>".
<br><br>

<u>Commands</u><br />
The <a href="https://steveproxna.blogspot.com/2009/12/command-design-pattern.html">Command</a> design pattern encapsulates a command request as an object.  In game code, an example of a command request could be a move command: e.g. move sprite left, right or jump command flip, swap etc.  
<br><br>
Coindentally, Lil EK uses 8x commands therefore each command stored as single bit mask; all 8x commands stored within a single byte.  This way, combos can be bitwised detected rather than flattened out individually.
<br />

<u>Sprites</u><br />
The Title screen uses <a href="https://github.com/sverx/devkitSMS/blob/master/SMSlib/src/SMSlib.h#L58">Zoom</a> sprites in which the 24x32 player sprite is magnified as a 48x64 sprite.  However, SMS I has a <a href="https://github.com/sverx/devkitSMS/wiki/Advanced">bug</a> in how the VDP processes zoomed sprites.   Therefore, to circumvent the VDP bug you have to draw 4x additional sprites on the same Y coordinates of the original sprites as seen in <a href="https://github.com/SteveProXNA/LilEvelKnievelSMS/blob/main/dev/screen/title_screen.c#L113">title_screen.c</a> file.

<u>Summary</u><br />
There has been progressive interest in Sega Master System Homebrew as coding competition entries become more impressive.  Unfortunately due to time constraints, the game mechanics and level design were not able to be fully developed at this time.  However the foundation for potential future development did receive some incredible <a href="https://www.smspower.org/forums/19526-CodingCompetition2023LilEvelKnievelBySteveProXNA">feedback</a> from the community and Sega 8-bit preservation and fanaticism fans.
