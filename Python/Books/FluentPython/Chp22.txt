# Chapter 22 – Async Programming with `asyncio` (Summary)

## 1. Main Idea of the Chapter

This chapter introduces **modern asynchronous programming in Python using `asyncio`**.

Key idea:

> `asyncio` enables **highly scalable concurrency** using **coroutines and an event loop**, without threads or processes.

It builds directly on:

* generators → Chapter 17
* coroutines → Chapter 19
* futures → Chapter 21

---

# 2. What Is `asyncio`?

`asyncio` is a library for writing **concurrent code using async/await syntax**.

Core components:

* **coroutines (`async def`)**
* **tasks**
* **event loop**
* **futures (internally)**

It is designed mainly for:

* **I/O-bound workloads**
* high-concurrency systems (e.g., web clients, servers)

---

# 3. Coroutines with `async` / `await`

Modern coroutines are defined with:

```python
async def fetch():
    ...
```

Inside them:

```python
await some_async_function()
```

Key behavior:

* `await` **suspends execution**
* control returns to the event loop
* another task can run

---

## 4. Event Loop

The **event loop** is the core of `asyncio`.

Responsibilities:

* schedules tasks
* runs coroutines
* waits for I/O events
* resumes paused tasks

Example:

```python
asyncio.run(main())
```

This:

1. creates an event loop
2. runs the coroutine
3. closes the loop when done

---

# 5. Tasks

A **Task** wraps a coroutine and schedules it for execution.

Create tasks using:

```python
asyncio.create_task(coro())
```

Example:

```python
async def main():
    t1 = asyncio.create_task(job1())
    t2 = asyncio.create_task(job2())
    await t1
    await t2
```

Tasks run **concurrently** within the event loop.

---

# 6. Concurrency with `asyncio`

Concurrency happens when coroutines:

* reach `await`
* yield control to the event loop

Example:

```python
await asyncio.sleep(1)
```

This:

* pauses the coroutine
* allows others to run

Important:

> No preemption — tasks must **cooperate**.

---

# 7. `await` vs Blocking Code

Critical rule:

❌ Blocking code:

```python
time.sleep(1)
```

✔ Async version:

```python
await asyncio.sleep(1)
```

Blocking operations:

* freeze the entire event loop
* prevent concurrency

---

# 8. Running Many Tasks

### Using `gather()`

```python
results = await asyncio.gather(*coroutines)
```

Characteristics:

* runs all coroutines concurrently
* returns results in order
* fails fast (propagates exceptions)

---

### Using `as_completed()`

```python
for coro in asyncio.as_completed(tasks):
    result = await coro
```

Advantages:

* results processed **as soon as ready**
* useful for progress tracking

---

# 9. Async Iteration

Async iterables use:

```python
__aiter__()
__anext__()
```

Used with:

```python
async for item in async_iterable:
```

Example use:

* streaming data
* async file/network reading

---

# 10. Async Context Managers

Async version of `with`:

```python
async with resource:
```

Uses:

```python
__aenter__()
__aexit__()
```

Example:

```python
async with session.get(url) as response:
    data = await response.text()
```

# 11. Case Study: Downloading Files (Async Version)

The chapter revisits the **flag downloader example** from Chapter 21.

Versions compared:

* sequential
* threads (`ThreadPoolExecutor`)
* async (`asyncio`)

Results:

* async version performs **very well**
* often comparable or better than threads
* uses fewer system resources

Reason:

* efficient handling of many I/O operations.

# 12. Error Handling in Async Code

Errors behave like normal exceptions.

Example:

```python
try:
    await task
except Exception:
    ...
```

With `gather()`:

* default: stops on first exception
* can use `return_exceptions=True` to collect all.

---

# 13. Timeouts and Cancellation

### Timeout:

```python
await asyncio.wait_for(coro, timeout=5)
```

### Cancellation:

```python
task.cancel()
```

Cancellation raises:

```python
asyncio.CancelledError
```

Tasks should handle cancellation cleanly.

---

# 14. Asyncio vs Threads vs Processes

| Feature         | Threads | Processes | Async    |
| --------------- | ------- | --------- | -------- |
| Parallel CPU    | ❌ (GIL) | ✅         | ❌        |
| I/O concurrency | ✅       | ✅         | ✅ (best) |
| Memory usage    | medium  | high      | low      |
| Complexity      | medium  | high      | medium   |

Key rule:

* CPU-bound → multiprocessing
* I/O-bound → asyncio (or threads)

---

# 15. Async Ecosystem

Common async libraries:

* `aiohttp` → HTTP client/server
* async DB drivers
* async web frameworks (FastAPI, etc.)

These are built on top of `asyncio`.

---

# 16. Design Philosophy

Async programming emphasizes:

### Explicit suspension points

Every `await` shows:

* where execution may pause

### Cooperative multitasking

Tasks must **yield control voluntarily**.

### Scalability

Async can handle **thousands of concurrent tasks** efficiently.

---

# 17. Key Takeaways

### 1️⃣ `asyncio` uses coroutines and an event loop

No threads required for concurrency.

---

### 2️⃣ `async` / `await` define asynchronous code

* `async def` → coroutine
* `await` → suspension point

---

### 3️⃣ Tasks enable concurrent execution

Use:

```python
asyncio.create_task()
```

---

### 4️⃣ Avoid blocking operations

Always use async equivalents.

---

### 5️⃣ `gather()` and `as_completed()` manage multiple tasks

* `gather()` → ordered results
* `as_completed()` → immediate results

---

### 6️⃣ Async is ideal for I/O-heavy workloads

Example:

* web requests
* APIs
* streaming data

---

### 7️⃣ Async builds on earlier concepts

```text
Iterators → Generators → Coroutines → Futures → asyncio
```
