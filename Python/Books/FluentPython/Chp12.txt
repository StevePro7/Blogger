2. Python Sequences Use Protocols (Duck Typing)

Python relies on protocols, not strict inheritance.

A protocol = an informal interface defined by expected methods.
If a class implements those methods, Python treats it as that type.

For sequences, the core protocol methods are:

__len__()
__getitem__()

Any class implementing them behaves like a sequence, even if it does not inherit from Sequence.

3. Building a Custom Sequence (Vector)

The implementation strategy:

Use composition rather than inheritance.

Example internal storage:

self._components = array('d', components)

The class wraps an array and exposes sequence behavior through special methods.

Benefits:

memory efficient

4. Implementing the Sequence Protocol
__len__

Returns number of items.

def __len__(self):
    return len(self._components)

This enables:

len(v)
__getitem__

Returns elements by index or slice.

def __getitem__(self, index):
    return self._components[index]

This enables:

indexing

slicing

iteration

in operator fallback

Python will try iteration using __getitem__ if __iter__ is missing.

5. Supporting Proper Slicing

Problem:

Default slicing of the underlying container returns the wrong type.

Example problem:

Vector slice -> array

But built-in sequences return same type:

list slice -> list
tuple slice -> tuple
Solution

Detect slice objects inside __getitem__.

Example logic:

if isinstance(index, slice):
    return Vector(self._components[index])
else:
    return self._components[index]

Now:

v[1:4] -> Vector

instead of returning the internal container.

6. How Python Represents Slices

When Python sees:

obj[1:4:2]

It actually calls:

obj.__getitem__(slice(1,4,2))

A slice object contains:

start
stop
step

Example:

slice(1,4,2)

The method slice.indices(length) normalizes values for a given sequence size.

8. Hashing Immutable Sequences

To make objects usable as dictionary keys or set elements:

__hash__()

Vector hashing combines the hashes of all components.

Example approach:

hashes = (hash(x) for x in components)
reduce(operator.xor, hashes)

Key principle:

Objects that compare equal must produce the same hash.

9. Faster Equality Comparison

__eq__ compares vectors element-by-element.

Efficient version:

Compare lengths first

Compare components

Example idea:

return len(self) == len(other) and all(a == b for a,b in zip(self, other))

11. Important Design Principles
1. Favor composition over inheritance

Instead of subclassing built-in types, wrap them.

2. Implement protocols, not class hierarchies

Behavior matters more than inheritance.

3. Minimal methods unlock many features

For sequences:

__len__
__getitem__

are usually enough.

4. Match built-in type behavior

Example:

slicing returns same type

hashing respects equality
