# Chapter 17 – Iterables, Iterators, and Generators (Summary)

## 1. Core Idea of the Chapter

Iteration is **fundamental to Python**. Many language features rely on iterators:

* `for` loops
* comprehensions (`list`, `set`, `dict`)
* unpacking (`a, b = ...`)
* file iteration
* functions like `sum`, `max`, `sorted`

Python integrates the **Iterator design pattern directly into the language**, so developers rarely need to implement it manually. 

---

# 2. Iterable vs Iterator

### Iterable

An **iterable** is any object that can return an iterator.

Examples:

```python
list
tuple
dict
set
str
range
```

An object is iterable if it:

* implements `__iter__()`
* or implements the **sequence protocol** (`__getitem__` with indexes).

---

### Iterator

An **iterator** is an object that produces values one at a time.

It must implement:

```python
__next__()
__iter__()
```

Behavior:

* `__next__()` → returns next item
* raises `StopIteration` when finished

Example use:

```python
it = iter(my_list)
next(it)
```

---

# 3. How Python Iteration Works

When Python runs a `for` loop:

```python
for x in obj:
    ...
```

Python internally performs:

1. `iter(obj)` → get iterator
2. repeatedly call `next(iterator)`
3. stop when `StopIteration` occurs

This mechanism is built into Python’s iteration protocol.

---

# 4. Why Sequences Are Iterable

Objects that implement the **sequence protocol** are automatically iterable.

Example:

```python
class MySeq:
    def __getitem__(self, index):
        ...
```

Python will keep calling `__getitem__(0)`, `__getitem__(1)`, etc., until an `IndexError`.

So even without `__iter__`, the object can still be iterable.

---

# 5. Implementing Iteration with `__iter__`

The recommended modern approach:

```python
def __iter__(self):
    return iterator
```

Example:

```python
class Sentence:
    def __iter__(self):
        return SentenceIterator(self.words)
```

This explicitly returns an iterator object.

---

# 6. The Classic Iterator Pattern

A manual iterator usually looks like:

```python
class MyIterator:

    def __init__(self, data):
        self.data = data
        self.index = 0

    def __next__(self):
        if self.index >= len(self.data):
            raise StopIteration
        item = self.data[self.index]
        self.index += 1
        return item

    def __iter__(self):
        return self
```

Important rule:

> Do **not make the iterable object its own iterator** (unless appropriate), because that prevents multiple independent iterations. 

---

# 7. Generator Functions

A **generator function** is the easiest way to create an iterator.

It uses the keyword:

```python
yield
```

Example:

```python
def count_up(n):
    i = 0
    while i < n:
        yield i
        i += 1
```

Calling the function returns a **generator iterator**.

Key property:

Each `yield`:

* pauses execution
* saves local state
* resumes later from the same point. 

---

# 8. Generator Expressions

Generator expressions are like **lazy list comprehensions**.

Example:

```python
(x*x for x in range(10))
```

Difference from list comprehension:

```python
[x*x for x in range(10)]
```

| List comprehension    | Generator expression   |
| --------------------- | ---------------------- |
| builds list in memory | produces values lazily |
| eager evaluation      | lazy evaluation        |

Generators are better for **large datasets or streams**.

---

# 9. Lazy Evaluation

Generators compute values **only when needed**.

Benefits:

* lower memory usage
* process large data streams
* efficient pipelines

Example use cases:

* reading huge files
* database records
* network streams

This is why generators are widely used in Python.

---

# 10. `yield from` – Delegating Generators

The `yield from` syntax allows one generator to **delegate iteration to another generator**.

Example:

```python
def gen():
    yield from other_gen()
```

Equivalent to:

```python
for item in other_gen():
    yield item
```

But `yield from` also:

* forwards values
* forwards exceptions
* forwards return values

This makes it **much more powerful**. 

---

# 11. Generator Functions in the Standard Library

Python’s `itertools` module contains many **generator-based tools**, such as:

* `count()`
* `cycle()`
* `repeat()`
* `chain()`
* `islice()`
* `accumulate()`

These allow powerful **data-processing pipelines**.

---

# 12. `iter()` Built-in Function

`iter()` has two forms.

### 1️⃣ Standard iterator creation

```python
iter(obj)
```

Returns iterator from iterable.

---

### 2️⃣ Callable iterator

```python
iter(function, sentinel)
```

Repeatedly calls `function()` until it returns `sentinel`.

Example pattern:

```python
for chunk in iter(read_block, ""):
    process(chunk)
```

---

# 13. Practical Case Study: Streaming Data Processing

The chapter presents a **database conversion utility** that uses generators.

Idea:

* one generator reads records
* another transforms data
* another writes output

Benefits:

* low memory usage
* modular pipeline
* easy to extend with new input formats. 

---

# 14. Generators vs Coroutines

Important conceptual distinction:

| Generators         | Coroutines               |
| ------------------ | ------------------------ |
| produce data       | consume data             |
| used for iteration | used for async workflows |

Generators:

```
pull model
```

Coroutines:

```
push model
```

Mixing them can be confusing, so they are discussed separately in later chapters. 

---

# 15. Key Takeaways

### Iteration is central to Python

Almost every data-processing feature uses iterators.

---

### Iterable ≠ Iterator

* iterable → can produce an iterator
* iterator → produces values

---

### Generators simplify iterator creation

Instead of writing complex iterator classes, use:

```
yield
```

---

### Generator expressions enable lazy computation

Efficient for **large datasets and streams**.

---

### `yield from` allows generator delegation

Useful for building **generator pipelines**.

---

### Generators enable memory-efficient data processing

Ideal for:

* big files
* database streams
* network data

---

**Core message of Chapter 17**

> Python integrates the Iterator pattern deeply into the language, and generators provide a simple, powerful way to process data lazily and efficiently.
