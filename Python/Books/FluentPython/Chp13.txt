# Chapter 13 – Interfaces, Protocols, and ABCs (Summary)

## 1. Core Idea of the Chapter

The chapter explains **how Python defines and uses interfaces** without requiring explicit declarations like many other languages.

The key principle:

> **In Python, a type is defined by the operations it supports (its interface), not by its class name.** 

Python supports **four complementary approaches to typing and interfaces**.

---

# 2. The Four Typing Approaches in Python

Python provides **four ways to describe interfaces and types**.

### 1️⃣ Duck Typing (Dynamic Protocols)

Python’s **original and default approach**.

Concept:

> “If it walks like a duck and quacks like a duck, it's a duck.”

Meaning:

* If an object **implements the required methods**, it can be used.

Example:

```python
len(obj)
for x in obj
obj[0]
```

If the object supports these operations, Python treats it like a sequence.

Characteristics:

* **Flexible**
* **No explicit interface declaration**
* **Checked at runtime**

---

### 2️⃣ Goose Typing (Abstract Base Classes)

Introduced with **Abstract Base Classes (ABCs)**.

Concept:

* Explicit interface defined by a **base class**.

Example:

```python
from collections.abc import Sequence

isinstance(obj, Sequence)
```

If a class inherits from an ABC, it **declares that it supports that interface**.

Purpose:

* runtime type checking
* clearer API contracts

This is called **goose typing** because objects must explicitly declare their type. 

---

### 3️⃣ Static Typing

Uses **type hints and static analysis tools**.

Example:

```python
def func(x: int) -> int:
```

Tools like **mypy** check types **before runtime**.

Key points:

* optional
* external tools perform checks
* more common in large projects

---

### 4️⃣ Static Duck Typing (Protocols)

Introduced with **`typing.Protocol` (PEP 544)**.

Concept:

* interface defined by **method signatures**
* no inheritance required

Example:

```python
from typing import Protocol

class SupportsClose(Protocol):
    def close(self) -> None: ...
```

Any class with `close()` satisfies the protocol.

Benefits:

* works with **static type checking**
* still follows **duck typing philosophy**

---

# 3. Dynamic Protocols (Duck Typing)

Protocols are **informal interfaces**.

A protocol simply means:

> A set of methods an object must provide to play a role. 

Example: **Sequence protocol**

Key methods:

```
__getitem__()
__len__()
```

If an object implements them, it behaves like a sequence.

Python even tries to help:

* If `__iter__` is missing, Python tries **iteration using `__getitem__`**.
* If `__contains__` is missing, Python performs a **sequential search**. 

This makes Python very **tolerant and flexible**.

---

# 4. Monkey Patching and Runtime Protocol Implementation

Protocols can even be implemented **after a class is defined**.

Example concept:

```python
Class.method = new_method
```

This is called **monkey patching**.

Uses:

* adapting third-party libraries
* quick fixes
* testing

But it should be used **carefully** because it can break expectations.

---

# 5. Defensive Programming and “Fail Fast”

Duck typing can cause **runtime errors if methods are missing**.

Better approach:

Instead of checking types:

```python
isinstance(obj, SomeType)
```

Use:

```
try:
    obj.method()
except AttributeError:
    ...
```

This approach:

* relies on behavior
* avoids unnecessary type checks
* **fails quickly when assumptions are wrong**

---

# 6. Goose Typing with Abstract Base Classes (ABCs)

ABCs provide **explicit interface definitions**.

Example:

```python
from collections.abc import Sequence
```

A class can:

1️⃣ **inherit from an ABC**

```python
class MySeq(Sequence):
```

2️⃣ **register as a virtual subclass**

```python
Sequence.register(MyClass)
```

Virtual subclasses:

* behave as subclasses
* do not inherit code

---

# 7. Important ABCs in `collections.abc`

Some key interfaces provided by Python:

### Basic collection ABCs

| ABC       | Required Method |
| --------- | --------------- |
| Iterable  | `__iter__`      |
| Container | `__contains__`  |
| Sized     | `__len__`       |

These are combined in:

```
Collection
```

---

### Main container types

| ABC      | Represents          |
| -------- | ------------------- |
| Sequence | ordered collections |
| Mapping  | dictionaries        |
| Set      | sets                |

Example hierarchy exists in `collections.abc`. 

---

# 8. Creating Your Own ABC

To define a new ABC:

```python
from abc import ABC, abstractmethod

class MyABC(ABC):

    @abstractmethod
    def method(self):
        pass
```

Subclasses **must implement the abstract methods**.

However:

> Creating ABCs is usually only needed for frameworks or libraries.

---

# 9. Static Protocols (typing.Protocol)

Protocols define **interfaces for static type checkers**.

Example:

```python
from typing import Protocol

class RandomPicker(Protocol):
    def pick(self) -> int: ...
```

Any class implementing `pick()` satisfies it.

Advantages:

* works without inheritance
* flexible like duck typing
* safer with static analysis

---

# 10. Runtime Checkable Protocols

Protocols can optionally be checked at runtime:

```python
from typing import runtime_checkable

@runtime_checkable
class MyProtocol(Protocol):
```

Then:

```python
isinstance(obj, MyProtocol)
```

will work.

---

# 11. Protocol Design Principles

Good protocols should be:

### Small

Minimal methods.

Example:
Many Go interfaces contain **only one method**.

### Extend Instead of Modify

If you need more functionality:

```
create a new protocol
```

rather than changing the old one.

---

# 12. Numeric ABCs vs Numeric Protocols

Python defines numeric ABCs:

```
Number
Complex
Real
Rational
Integral
```

But these **do not work well with static typing tools**.

Instead, the `typing` module provides protocols like:

```
SupportsFloat
SupportsComplex
```

These are more compatible with static type checkers.

---

# 13. Key Takeaways

### Python has **four ways to define interfaces**

1. **Duck typing** (dynamic protocols)
2. **Goose typing** (ABCs)
3. **Static typing** (type hints)
4. **Static duck typing** (`typing.Protocol`)

---

### Protocols define behavior

Objects satisfy an interface by **implementing methods**, not by inheriting a class.

---

### ABCs are useful for

* runtime type checking
* formal interfaces
* framework design

---

### Protocols are best for

* static type checking
* flexible interfaces

---

**Main Message of the Chapter**

> Python provides multiple complementary ways to define interfaces, and understanding when to use duck typing, ABCs, or protocols is key to writing flexible and maintainable Python code.
