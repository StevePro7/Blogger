# Chapter 25 – Attribute Descriptors (Summary)

## 1. Main Idea of the Chapter

Descriptors are a **core mechanism behind attribute access in Python**.

Key idea:

> A **descriptor is an object that controls how attributes are accessed, set, or deleted**.

They are the foundation of:

* `property`
* methods
* class attributes
* ORMs
* many framework features

---

# 2. What Is a Descriptor?

A descriptor is any object that implements one or more of these methods:

```python
__get__(self, instance, owner)
__set__(self, instance, value)
__delete__(self, instance)
```

Example:

```python
class Descriptor:
    def __get__(self, instance, owner):
        ...
```

---

# 3. Descriptor Protocol

### `__get__`

* called when attribute is **read**

### `__set__`

* called when attribute is **assigned**

### `__delete__`

* called when attribute is **deleted**

---

# 4. Descriptor Types

## 1️⃣ Data Descriptors

Implement:

```python
__set__  (and/or __delete__)
```

Characteristics:

* **override instance attributes**
* always take priority

---

## 2️⃣ Non-Data Descriptors

Implement only:

```python
__get__
```

Characteristics:

* can be **overridden by instance attributes**

---

# 5. Attribute Lookup Order

This is one of the most important concepts.

When accessing:

```python
obj.attr
```

Python looks in this order:

1. **Data descriptor** in class
2. Instance `__dict__`
3. Non-data descriptor in class
4. Class attributes
5. Parent classes

Key rule:

> Data descriptors have the **highest priority**.

---

# 6. Example: Managed Attribute

```python
class Quantity:
    def __set__(self, instance, value):
        if value <= 0:
            raise ValueError("must be > 0")
        instance.__dict__[self.name] = value
```

Used in:

```python
class Product:
    weight = Quantity()
```

This enforces **validation logic automatically**.

---

# 7. `__set_name__`

Called when the descriptor is assigned to a class attribute.

```python
def __set_name__(self, owner, name):
    self.name = name
```

Benefits:

* descriptor knows its attribute name
* avoids hardcoding names

---

# 8. Storage Strategy

Descriptors typically store values in:

```python
instance.__dict__
```

Using:

```python
instance.__dict__[self.name]
```

This avoids infinite recursion.

---

# 9. Overriding vs Non-Overriding Descriptors

### Overriding (Data Descriptor)

```python
__get__ + __set__
```

* always controls access
* cannot be bypassed by instance attributes

---

### Non-Overriding

```python
__get__ only
```

* instance attribute can override it

Example:

```python
obj.method = something_else
```

---

# 10. Methods Are Descriptors

Functions defined in classes are actually **non-data descriptors**.

Example:

```python
class A:
    def f(self):
        pass
```

Accessing:

```python
a.f
```

Triggers:

```python
f.__get__(a, A)
```

This creates a **bound method**.

---

# 11. Built-in Descriptors

Many Python features use descriptors:

### `property`

```python
@property
def x(self):
    return self._x
```

### Functions (methods)

* become bound methods via descriptors

### Class methods

```python
@classmethod
```

### Static methods

```python
@staticmethod
```

All implemented using descriptor protocol.

---

# 12. Validation with Descriptors

Descriptors are commonly used for:

* type checking
* range validation
* enforcing constraints

Example:

```python
if not isinstance(value, int):
    raise TypeError
```

---

# 13. Reusable Descriptor Classes

You can build reusable components:

```python
class Positive:
    def __set__(self, instance, value):
        if value <= 0:
            raise ValueError
```

Then reuse across classes.

---

# 14. Common Pitfalls

### ❌ Infinite recursion

Wrong:

```python
self.attr = value
```

Correct:

```python
instance.__dict__[self.name] = value
```

---

### ❌ Forgetting `__set_name__`

Leads to missing attribute names.

---

### ❌ Misunderstanding lookup order

Leads to confusing bugs.

---

# 15. Descriptor vs Property

| Descriptor    | Property              |
| ------------- | --------------------- |
| reusable      | tied to one attribute |
| more flexible | simpler               |
| more complex  | easier to use         |

Use:

* `property` → simple cases
* descriptors → reusable logic

---

# 16. Real-World Applications

Descriptors are used in:

* ORMs (Django, SQLAlchemy)
* validation libraries
* data models
* frameworks
* caching systems

---

# 17. Key Takeaways

### 1️⃣ Descriptors control attribute access

They define how attributes behave.

---

### 2️⃣ Three core methods

```python
__get__  __set__  __delete__
```

---

### 3️⃣ Data descriptors override instance attributes

They have highest priority.

---

### 4️⃣ Attribute lookup order is critical

Understanding it avoids bugs.

---

### 5️⃣ Methods and properties are descriptors

Descriptors are deeply built into Python.

---

### 6️⃣ Descriptors enable reusable logic

Especially for validation and computed attributes.

> Descriptors are a fundamental mechanism in Python that power attribute access and enable reusable, customizable behavior, forming the backbone of many advanced features and frameworks.
