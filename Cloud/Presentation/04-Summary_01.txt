Quick Summary of Definitions
<br />
  
1. Docker
<br />
<table width="99%" border="1">
<tr>
<td>
&nbsp;<span style="font-family: 'Wingdings';">•</span>&nbsp;
Platform as a Service product that uses OS-level virtualization to deliver S/W in packages = containers
<br />
&nbsp;<span style="font-family: 'Wingdings';">•</span>&nbsp;
Containers are isolated from one another and bundle their own S/W, libraries, and configurations file
<br />
&nbsp;<span style="font-family: 'Wingdings';">•</span>&nbsp;
Containers share signle OS kernel on host thus use fewer resources than Virtual Machines
</td>
</tr>
</table>
<br />
  
2. Kubernetes
<br />
<table width="99%" border="1">
<tr>
<td>
&nbsp;<span style="font-family: 'Wingdings';">•</span>&nbsp;
Open-source container orchestration system for automating application deployment, scaling and management
<br />
&nbsp;<span style="font-family: 'Wingdings';">•</span>&nbsp;
Runs containerized applications in cluster of host machines from container images typically built using Docker
</td>
</tr>
</table>
<br />  
  
3. Helm
<br />
<table width="99%" border="1">
<tr>
<td>
&nbsp;<span style="font-family: 'Wingdings';">•</span>&nbsp;
Kubernetes package manager to simplify process of managing + deploying applications to clusters via "Charts"
<br />
&nbsp;<span style="font-family: 'Wingdings';">•</span>&nbsp;
Helm facilitates configuration separated out in Values files and scaled out across environments: DEV, UAT, Prod
</td>
</tr>
</table>
